# 精子形态筛选代码编写方案

## 0. 本阶段目标

当前只做“精子筛选”的形态学部分，不做视频运动筛选，不使用 `SpermTracking/tracking` 中的无标签连续帧作为主要输入。第一版目标是：从已有图像和帧级位置标注中裁剪精子候选区域，分割精子头部，提取形态学量化指标，并按“拟合优度、长轴比、均匀度”进行评分和分级。

结合 Word 文档中的研究思路，本阶段应放在后续实时跟踪、吸取控制之前：先用形态学标准剔除结构缺失、头部异常、严重重叠、图像质量差的候选精子，为后续运动评价和吸取控制提供更可靠的目标。

## 1. 现有数据情况

已确认的数据结构如下：

```text
大创/
  nk/
  SpermTracking/
    ImagesWithLabels/
      images/
        0000.jpg
        0001.jpg
        ...
      labels/
        0000.txt
        0001.txt
        ...
    tracking/
      01/
      02/
      ...
```

`ImagesWithLabels/images/0000.jpg` 示例图像大小为 `870 x 810`，灰度图。`labels/0000.txt` 的每行格式不是 YOLO 归一化格式，而是：

```text
id x1 y1 x2 y2
```

例如：

```text
0 541.3199462890625 108.08777618408203 550.4077758789062 123.01619720458984
```

含义建议理解为：`id` 是该帧中的目标编号，`x1 y1 x2 y2` 是绝对像素坐标形式的候选框。由于标注的是位置框，不是精子头部 mask，所以第一版不能直接训练高质量分割模型，应先写传统图像处理流程，从候选框中裁剪 ROI，再做头部分割和形态测量。

## 2. 总体技术路线

第一版采用“弱标注 ROI + 传统图像处理 + 可解释评分”的路线：

1. 读取图像和同名标签文件。
2. 根据 `x1 y1 x2 y2` 裁剪每个候选精子 ROI，并在候选框周围增加边缘留白。
3. 对 ROI 做灰度归一化、去噪、背景校正、对比度增强。
4. 在 ROI 内分割精子头部，尽量排除尾部、背景颗粒、重叠精子。
5. 从头部二值 mask 和轮廓中提取形态指标：头部长度 L、头部宽度 W、长宽比 R、短轴对称 SAS、长轴对称 LAS、头部面积 HA、头维 HD。
6. 计算评分维度：拟合优度、长轴比、均匀度。
7. 输出每个候选精子的特征表、分级结果、可视化图。
8. 人工抽查一批结果，调整阈值。

后续如果手动补充 50 到 200 个精子头部 mask，可以升级为 U-Net 或 Mask R-CNN 分割；但在当前标注较少的条件下，先写传统流程更稳。

注意：`axis_ratio_target: 1.62` 是参考人类正常精子头部长宽比的大致中心值。你们数据里有小鼠、猪、人类或未知来源时，绝对阈值不能直接混用。第一版可以先用像素相对指标跑通流程，后续根据物种和显微镜标定调整。

## 5. 第二步：读取图像和标签

在 `dataset.py` 中实现：

1. 遍历 `image_dir` 下所有 `.jpg`。
2. 找同名 `.txt` 标签，例如 `0000.jpg` 对应 `0000.txt`。
3. 如果标签文件不存在，记录到 `failed_cases.csv`。
4. 如果标签文件为空，跳过该图。
5. 对每一行按空格切分，解析为：

```python
target_id = int(parts[0])
x1, y1, x2, y2 = map(float, parts[1:5])
```

6. 将浮点坐标转为裁剪坐标时不要立刻四舍五入到太小，建议：

```python
x1 = floor(x1)
y1 = floor(y1)
x2 = ceil(x2)
y2 = ceil(y2)
```

7. 坐标必须 clamp 到图像范围内：

```python
x1 = max(0, x1)
y1 = max(0, y1)
x2 = min(image_width - 1, x2)
y2 = min(image_height - 1, y2)
```

输出一个统一的数据结构：

```python
{
    "image_path": ".../0000.jpg",
    "label_path": ".../0000.txt",
    "image_id": "0000",
    "target_id": 0,
    "bbox": [x1, y1, x2, y2],
}
```

## 6. 第三步：裁剪 ROI

因为标签框通常只框住精子头部或局部，裁剪时要扩展边距，否则后续很难判断尾部连接、重叠、背景颗粒。`crop.py` 中实现：

1. 输入整图、bbox、`margin_px`。
2. 用 bbox 中心和宽高决定 ROI。
3. 在四周增加 `margin_px`，默认 16 像素。
4. 如果 bbox 太小，比如宽或高小于 4 像素，直接标记失败。
5. 保存 ROI 与原图坐标映射关系。

输出：

```python
{
    "roi": roi_image,
    "roi_bbox_global": [rx1, ry1, rx2, ry2],
    "target_bbox_local": [x1-rx1, y1-ry1, x2-rx1, y2-ry1],
}
```

这个映射很重要，因为最后可视化时要把头部轮廓和椭圆画回原图。

## 7. 第四步：图像预处理

在 `preprocess.py` 中实现一个函数：

```python
def preprocess_roi(roi_gray, config):
    ...
    return enhanced_gray
```

建议处理顺序：

1. 确保 ROI 是单通道灰度图。
2. 做归一化，把灰度拉到 `0-255`。
3. 用 Gaussian 或 median blur 去噪。
4. 做背景校正：
   - 用较大的 Gaussian blur 得到背景图。
   - 原图减背景图，再重新归一化。
5. 用 CLAHE 增强局部对比度。

为什么要做背景校正：精子图像存在显微照明不均、背景灰度缓慢变化、局部杂质，直接阈值分割容易把背景当成头部。

## 8. 第五步：头部分割方法

当前推荐先做传统分割，不要马上训练深度模型。

### 8.1 第一版传统分割流程

在 `segment_head.py` 中写：

```python
def segment_head(roi_gray, target_bbox_local, config):
    ...
    return head_mask, quality_info
```

流程：

1. 在 ROI 内优先关注 `target_bbox_local` 附近。
2. 用 Otsu 或 adaptive threshold 得到候选二值图。
3. 如果精子头部在增强图中偏亮，就保留亮区域；如果偏暗，就尝试反相阈值，两种结果都跑一遍，选质量更高者。
4. 做形态学开运算，去掉孤立噪声。
5. 做形态学闭运算，补齐头部内部小孔。
6. 找所有连通域或轮廓。
7. 对每个候选区域计算：
   - 面积是否在范围内；
   - 是否靠近标签框中心；
   - 是否能拟合椭圆；
   - 椭圆长宽比是否合理；
   - 轮廓是否过于细长，避免把尾巴当头部。
8. 选择综合质量最高的连通域作为头部 mask。
9. 如果没有合格连通域，返回失败原因。

### 8.2 头尾连接时的处理

精子尾部很细，可能和头部连在一起。可尝试：

1. 对二值图做一次轻微腐蚀，细尾会断开。
2. 找较圆、较大的连通域作为头部。
3. 再对头部区域轻微膨胀，恢复边界。

如果尾巴仍然连着，判断为低质量分割，写入 `reject_reason = "tail_connected"`，不要强行给高分。

### 8.3 重叠精子处理

如果一个 ROI 内出现多个头部或轮廓严重粘连：

1. 如果候选区域面积远大于正常范围，标记 `overlap_or_debris`。
2. 如果拟合椭圆 IoU 很低，标记 `bad_fit`。
3. 如果轮廓凸性差、凹陷多，标记 `irregular_or_overlap`。

第一版筛选宁愿保守剔除，不要把重叠个体评为优质精子。

## 9. 第六步：形态特征计算

在 `features.py` 中从 `head_mask` 和 `roi_gray` 计算特征。

### 9.1 轮廓和椭圆

1. 用 `findContours` 找头部 mask 最大外轮廓。
2. 如果轮廓点数小于 5，不能 `fitEllipse`，直接失败。
3. 拟合椭圆，得到：
   - 中心 `(cx, cy)`
   - 长轴 `major_axis`
   - 短轴 `minor_axis`
   - 角度 `theta`

定义：

```text
L = major_axis
W = minor_axis
R = L / W
HA = mask 中前景像素数量
HD = sqrt(4 * HA / pi)
```

这里 `HD` 是“头维”的第一版定义，也可理解为等效圆直径。如果你们老师或 Word 文档中对“头维”有更具体定义，后续只需要替换 `compute_head_dimension()` 这个函数。

如果有显微镜标定：

```text
L_um = L_px * um_per_pixel
W_um = W_px * um_per_pixel
HA_um2 = HA_px2 * um_per_pixel * um_per_pixel
HD_um = HD_px * um_per_pixel
```

如果没有标定，全部保留像素单位，不要伪造微米单位。

### 9.2 短轴对称 SAS 和长轴对称 LAS

先把 head mask 旋转到椭圆坐标系：

```text
x 方向 = 头部长轴方向
y 方向 = 头部短轴方向
```

建议定义：

```text
LAS = 关于长轴的镜像对称性
SAS = 关于短轴的镜像对称性
```

计算方式用镜像 IoU：

```text
LAS = IoU(mask, reflect(mask, about_long_axis))
SAS = IoU(mask, reflect(mask, about_short_axis))
```

范围为 `0-1`，越接近 1 越对称。

如果实现镜像 IoU 麻烦，第一版可用面积差近似：

```text
LAS = 1 - abs(area_upper - area_lower) / (area_upper + area_lower + eps)
SAS = 1 - abs(area_left - area_right) / (area_left + area_right + eps)
```

但最终建议用镜像 IoU，因为它不仅比较面积，还比较形状位置。

### 9.3 拟合优度

拟合优度用于衡量头部是否接近光滑椭圆。推荐用“头部 mask 和拟合椭圆 mask 的 IoU”：

```text
fit_iou = area(head_mask ∩ ellipse_mask) / area(head_mask ∪ ellipse_mask)
```

也可以额外计算轮廓残差：

```text
contour_residual = mean(distance(contour_points, fitted_ellipse_boundary)) / L
```

第一版只用 `fit_iou` 即可，简单、可解释、方便调参。

### 9.4 长轴比

用户提出的指标中已有长宽比 `R`，评分标准里还有“长轴比”。第一版建议将它定义为头部椭圆长短轴比：

```text
axis_ratio = L / W
```

然后用目标区间评分。若按人类精子头部形态参考，可先使用 `1.3-2.0` 的宽松范围，中心约 `1.6`；如果是小鼠或猪精子，需要重新标定。更稳妥的方法是从人工确认的正常样本中统计：

```text
R_target = median(R_normal_samples)
R_tolerance = 1.5 * IQR(R_normal_samples)
```

### 9.5 均匀度

均匀度建议同时考虑灰度均匀和形状均匀。

灰度均匀度：

```text
gray_uniformity = 1 - std(gray_values_in_head) / (mean(gray_values_in_head) + eps)
```

形状均匀度可先不单独做，后续可以加入：

```text
shape_uniformity = fit_iou * min(LAS, SAS)
```

第一版综合均匀度：

```text
uniformity = 0.6 * gray_uniformity + 0.4 * min(LAS, SAS)
```

如果图像中头部内部有明显空泡、缺口、亮暗不均，`gray_uniformity` 会下降；如果头部偏斜、局部缺损，`SAS/LAS` 会下降。

## 10. 第七步：评分与分级

在 `scoring.py` 中把每个维度归一化到 `0-100`。

### 10.1 拟合优度评分

配置：

```text
fit_iou_good = 0.85
fit_iou_bad = 0.60
```

计算：

```text
fit_score = clamp((fit_iou - fit_iou_bad) / (fit_iou_good - fit_iou_bad), 0, 1) * 100
```

含义：

| fit_iou | 解释 |
| --- | --- |
| >= 0.85 | 头部接近光滑椭圆 |
| 0.60-0.85 | 可疑或轻微异常 |
| < 0.60 | 形状不规则、重叠、分割失败风险高 |

### 10.2 长轴比评分

配置：

```text
R_target = 1.62
R_tolerance = 0.35
```

计算：

```text
axis_score = clamp(1 - abs(R - R_target) / R_tolerance, 0, 1) * 100
```

后续更推荐用人工正常样本统计出的 `R_target`，而不是固定写死。

### 10.3 均匀度评分

配置：

```text
uniformity_good = 0.85
uniformity_bad = 0.60
```

计算：

```text
uniformity_score = clamp(
    (uniformity - uniformity_bad) / (uniformity_good - uniformity_bad),
    0,
    1
) * 100
```

### 10.4 总分

评分标准为三维：

```text
total_score =
    0.40 * fit_score +
    0.30 * axis_score +
    0.30 * uniformity_score
```

同时加入硬性剔除条件：

```text
如果分割失败：Reject
如果 HA 太小或太大：Reject
如果 L/W 极端异常：Reject
如果 min(SAS, LAS) < 0.55：Reject
如果 fit_iou < 0.50：Reject
```

### 10.5 分级

建议第一版分为：

| 等级 | 分数 | 含义 |
| --- | --- | --- |
| A | `>= 85` | 形态优，优先进入后续运动评价 |
| B | `70-85` | 形态基本正常，可作为备选 |
| C | `55-70` | 可疑，建议人工复核 |
| D | `< 55` | 形态较差，不建议进入后续流程 |
| Reject | 硬性失败 | 分割失败、重叠、严重异常、图像质量差 |

## 12. 第九步：批量运行脚本

在 `scripts/run_morphology_screening.py` 中写入口：

```python
def main():
    config = load_config("configs/morphology.yaml")
    samples = load_dataset(config)

    rows = []
    for sample in samples:
        image = read_image(sample["image_path"])
        roi_info = crop_roi(image, sample["bbox"], config)
        roi_pre = preprocess_roi(roi_info["roi"], config)
        mask, quality = segment_head(roi_pre, roi_info["target_bbox_local"], config)

        if mask is None:
            rows.append(make_failed_row(sample, quality))
            continue

        features = compute_features(mask, roi_pre, config)
        scores = score_features(features, config)
        save_overlay(image, roi_info, mask, features, scores, config)
        rows.append(make_success_row(sample, features, scores))

    save_csv(rows, "outputs/morphology_scores.csv")
```

先保证这个脚本能跑完整个 `ImagesWithLabels`，哪怕一开始分割效果一般，也要能输出失败原因。

## 13. 第十步：人工复核和调阈值

因为现在没有大量精子形态等级标签，必须加入人工复核环节。

建议流程：

1. 随机抽取 100 个候选 ROI。
2. 将 overlay 图按 `A/B/C/D/Reject` 分类保存。
3. 人工打开查看：
   - A 里面是否有明显异常；
   - Reject 里面是否有明显正常；
   - C 是否主要是边界样本。
4. 记录错误类型：
   - 头部分割偏小；
   - 把尾部当成头部；
   - 两个精子粘连；
   - 背景颗粒误检；
   - 灰度不均导致空泡误判；
   - bbox 太小导致 ROI 缺失。
5. 根据错误类型调：
   - crop margin；
   - 阈值方法；
   - 面积范围；
   - `fit_iou_good/bad`；
   - `axis_ratio_target/tolerance`；
   - `uniformity_good/bad`。

不要一开始追求模型复杂，先把“失败原因可解释、结果可复核”做好。

## 14. 后续升级路线

### 14.1 补充 mask 标注

如果传统分割在复杂背景下不稳定，建议手工标注一小批精子头部 mask：

1. 从不同图像、不同亮度、不同密度场景中选 ROI。
2. 标注 50 到 100 个正常/异常头部 mask。
3. 保留 20% 做验证集。
4. 训练轻量 U-Net。
5. 用 U-Net 输出 head mask，再复用当前的 `features.py` 和 `scoring.py`。

这样可以保留当前评分系统，不需要推翻代码。

### 14.2 引入异常类型分类


### 14.3 与运动筛选衔接

当前不使用视频。后续衔接时：

1. 形态评分给每个目标一个先验质量分。
2. 跟踪算法计算运动速度、轨迹稳定性、摆动情况。
3. 最终选择：

```text
final_score = morphology_weight * morphology_score + motility_weight * motility_score
```

这样符合 Word 文档中“先形态筛选，再实时跟踪，再吸取控制”的整体路线。

## 15. 本阶段最小可行版本

最小版本只需要完成以下 8 件事：

1. `dataset.py` 能读图像和标签。
2. `crop.py` 能裁剪 ROI。
3. `preprocess.py` 能增强 ROI。
4. `segment_head.py` 能输出头部 mask 或失败原因。
5. `features.py` 能算 L、W、R、SAS、LAS、HA、HD、fit_iou、uniformity。
6. `scoring.py` 能输出 A/B/C/D/Reject。
7. `batch_run.py` 能生成 `morphology_scores.csv`。
8. `visualize.py` 能生成 overlay 供人工复核。

先做到这一步，就已经可以支撑“精子形态学量化分级”的阶段性研究和汇报。
